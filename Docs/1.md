1. Architectural System Overview

Due to hardware-level EGL driver abstraction mismatches (libEGL warning: egl: failed to create dri2 screen) commonly found in hybrid graphics configurations running Ubuntu 24.04 with modern vendor-packaged simulators, this system architecture isolates the visual processing thread from the computational simulation layout.

The environment is configured to execute using Headless Server Mode. The orchestration framework binds a custom static landscape description (SDF), an abstracted system-wide kinematic descriptor package (URDF), and a centralized automation coordinator (Launch Script).

                              +---------------------------------------+
                              |      Gazebo Simulation Server         |
                              |   (Loads room_world.sdf via -s Headless)
                              +-------------------+-------------------+
                                                  |

  +-------------------------------+         +-----+-----+         +--------------------------+

3.3 Central Automation Controller: simulation.launch.py

This orchestration script dynamically links the target workspace parameters together. It resolves package shares, maps out the exact robot configuration details out of the core system path, launches the Gazebo simulation runtime background processes using native environment calls, spins up the spatial frame coordinate transform broadcaster (robot_state_publisher), and triggers an absolute entry injector spawner node (create).

    Absolute Target System Path: /home/koko/Projects/orangewood-training/amr/src/room_simulator/launch/simulation.launch.py

Python

import os
from ament_index_python.packages import get_package_share_directory
from launch import LaunchDescription
from launch.actions import IncludeLaunchDescription
from launch.launch_description_sources import PythonLaunchDescriptionSource
from launch_ros.actions import Node

def generate_launch_description():
    # 1. Resolve standard internal package share directories
    room_simulator_share = get_package_share_directory('room_simulator')
    turtlebot3_desc_share = get_package_share_directory('turtlebot3_description')
    ros_gz_sim_share = get_package_share_directory('ros_gz_sim')

    # Define absolute target path to the system custom environment landscape file
    world_file_path = '/home/koko/Projects/orangewood-training/amr/src/room_simulator/worlds/room_world.sdf'
    
    # Establish direct target mapping to the globally tracked robot URDF definition
    urdf_file_path = os.path.join(turtlebot3_desc_share, 'urdf', 'turtlebot3_burger.urdf')

    # Extract the internal text mapping vectors out of the target URDF definition
    with open(urdf_file_path, 'r') as infp:
        robot_description_content = infp.read()

    # 2. Declare orchestration node for the background simulation engine launcher
    gazebo_launch = IncludeLaunchDescription(
        PythonLaunchDescriptionSource(
            os.path.join(ros_gz_sim_share, 'launch', 'gz_sim.launch.py')
        ),
        # Pass the background run arguments out directly to the launch instance targets
        launch_arguments={'gz_args': f'-r {world_file_path}'}.items()
    )

    # 3. Initialize state publisher to calculate kinematic joint frame links (/tf)
    robot_state_publisher = Node(
        package='robot_state_publisher',
        executable='robot_state_publisher',
        output='screen',
        parameters=[{
            'robot_description': robot_description_content, 
            'use_sim_time': True
        }]
    )

    # 4. Initialize Entity Spawner client task to construct the physical robot instance
    robot_spawner = Node(
        package='ros_gz_sim',
        executable='create',
        output='screen',
        arguments=[
            '-name', 'turtlebot3_burger',
            '-topic', 'robot_description',
            '-x', '0.0',
            '-y', '0.0',
            '-z', '0.1'
        ]
    )

    return LaunchDescription([
        gazebo_launch,
        robot_state_publisher,
        robot_spawner
    ])
